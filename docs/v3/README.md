# Flux v3 — 文档索引

> **Flux is a deterministic execution kernel for DAG-based workflows.**
> It does not reason, it does not own users, and it does not manage conversations.
> It executes plans generated by Agent Runtime.
>
> v3 是一次架构重新定位：flux 从 "Tool-First Agent Runtime" 重新定位为 **"Agent 的 Workflow Engine"**。

## 核心文档

- **[architecture-v3.md](architecture-v3.md)** — 五层架构、持久化端口、集成契约、删减清单、设计原则、路线图、决策记录
- **[development-plan.md](development-plan.md)** — 实施计划：5 个 Phase，预计 10-16 天，全部可在 DreamAI 当前架构中独立完成
- **[code-agent-brief.md](code-agent-brief.md)** — 给 code-agent 团队的架构对齐建议：ConversationStore 协议化、与 flux 的关系、不需要做的事

## 核心变更（v2 → v3）

1. **五层架构**：新增 Persistence Layer 作为第一等架构层次
2. **Store 抽象**：ConversationStore / WorkflowStore / AwaitStore / TraceStore 是接口，SQLite / PG / Memory 是 Adapter
3. **Agent Runtime 与 Workflow Runtime 分离**：LLMPlanner 删除，增量 Agent Loop 统一由 Agent Runtime 提供
4. **flux 重新定位**：从 "Agent Runtime" 变为 "Agent 的 Workflow Engine"，通过 Tool 协议集成

## 与 v2 的关系

v3 不是 v2 的重写。v2 的核心设计（PlanSource、Scheduler、definition-free runtime、工具=MCP）全部保留。v3 做的是：

1. 把 Agent Runtime 的职责上移到独立层
2. 把 flux 重新聚焦到 Workflow Engine
3. 新增 Persistence Layer，Store 接口与数据库解耦
4. 用 Tool 协议替代库依赖做集成

v2 文档仍然有效，保留为历史参考。冲突时以 v3 为准。
